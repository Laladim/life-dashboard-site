Lesson 6 - Troubleshooting skills

Learning objectives (from the course)

  • Use the skills validator to catch structural issues before debugging
  • Diagnose and fix common skill triggering and loading problems
  • Resolve skill priority conflicts between enterprise, personal, project, and plugin skills
  • Debug runtime errors including missing dependencies, permissions, and path issues

The lesson in plain English

When a skill misbehaves it falls into one of a few categories. Start with the agent skills validator (install via uv; run from the skill dir or anywhere) - it catches structural problems before you waste time elsewhere.

  • Doesn't trigger → almost always the description. Claude uses semantic matching; your request must overlap the description's meaning. Add trigger phrases users would actually say; test variations ("help me profile this," "why is this slow?", "make this faster"); add any that fail to the description.
  • Doesn't load → structure. The SKILL.md must be inside a named directory (not the skills root), and the filename must be exactly SKILL.md (caps SKILL, lowercase md). Run claude --debug to see loading errors mentioning your skill name.
  • Wrong skill used → descriptions too similar. Make them distinct and specific.
  • Priority conflict / being shadowed → an enterprise (or higher-priority) skill has the same name and wins. Fix: rename yours to something distinct (easier), or talk to your admin.
  • Plugin skills not appearing → clear the cache, restart Claude Code, reinstall. Still missing → plugin structure is likely wrong (validator earns its keep here).
  • Runtime errors (loads but fails) → missing dependencies (install them; note them in the description), permission issues (chmod +x scripts), and path separators (use forward slashes everywhere, even on Windows).

Quick checklist: not triggering → improve description/trigger phrases; not loading → check path, filename, YAML syntax; wrong skill → make descriptions distinct; shadowed → check priority, rename; plugins missing → clear cache + reinstall; runtime failure → check deps, permissions, paths.

Lesson 2 - Creating your first skill

Learning objectives (from the course)

  • Create a skill from scratch with proper frontmatter structure
  • Test and verify that a skill loads correctly in Claude Code
  • Explain how Claude Code matches incoming requests to available skills
  • Describe the skill priority hierarchy (Enterprise, Personal, Project, Plugins)

The lesson in plain English

Build a skill = make a directory whose name matches the skill, put a SKILL.md inside. Frontmatter has name (identifies it) and description (the matching criteria). Everything after the closing --- is the instructions Claude follows.

The walkthrough builds a personal pr-description skill in ~/.claude/skills/pr-description/. Then: restart Claude Code (skills load at startup), verify it appears in the skills list, and test it ("write a PR description for my changes").

How matching works under the hood: at startup Claude scans four locations (enterprise, personal, project, plugins) and loads only the name + description of each skill, not the full content. When you send a request, Claude compares it to those descriptions via semantic matching ("explain what this function does" matches a skill described as "explain code with visual diagrams" because intent overlaps). On a match it asks you to confirm loading the skill - keeping you aware of what context it's pulling - then reads the full SKILL.md and follows it.

Priority for name conflicts (memorize): Enterprise → Personal → Project → Plugins. Enterprise (managed settings) wins; plugins lose. This lets orgs enforce standards while individuals still customize via differently named skills. To avoid conflicts, use descriptive names (frontend-pr-review, not review).

Maintenance: edit SKILL.md to update; delete the directory to remove; restart Claude Code for changes to take effect.

Key takeaways (verbatim from course companion)

  • A skill is a directory containing a SKILL.md file with metadata (name, description) in frontmatter and instructions below
  • Claude loads only skill names and descriptions at startup, then matches incoming requests against those descriptions using semantic matching
  • You get a confirmation prompt before Claude loads the full skill content into context
  • Priority for name conflicts: Enterprise → Personal → Project → Plugins
  • To update a skill, edit its SKILL.md. To remove one, delete its directory. Always restart Claude Code for changes to take effect

Exam watch-outs (CCA-F, Domain 3)

  • Startup loads name + description ONLY - the full body loads on match. This is why skills are context-cheap, and it's flagged "important later."
  • Priority order is testable: Enterprise > Personal > Project > Plugins.
  • Restart required after any skill change - a likely "why isn't my skill showing?" distractor.
  • Directory name should match the skill name; the file must be exactly SKILL.md.

Terms introduced

  • Frontmatter instructions split = frontmatter (name/description) above ---, instructions below.
  • Semantic matching = Claude matches by meaning overlap between your request and the description, not exact keywords.
  • Confirmation step = the prompt before the full skill loads, so you stay aware of context.
  • Priority hierarchy = Enterprise → Personal → Project → Plugins (conflict resolution by same name).

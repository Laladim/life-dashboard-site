CCA Companion Source Hierarchy

Use this file when sources conflict.

Authority Order

  1. Official CCA-F exam guide PDF
  • Governs exam scope, domains, task areas, scenarios, sample-question style, readiness standard, and out-of-scope topics.
  • If it conflicts with any local note, the guide wins.
  1. CCA trainer operating files
  • knowledge-base.md is a derived digest, not the primary authority.
  • guide-mastery-matrix.md governs coverage tracking.
  • readiness-gates.md governs whether Shela can be called ready.
  • progress.md and learning-log.md govern what has actually been practiced.
  1. Official Anthropic sources for current Claude product truth
  • Anthropic platform release notes: https://platform.claude.com/docs/en/release-notes/overview
  • Claude app release notes: https://support.claude.com/en/articles/12138966-release-notes
  • Claude Code changelog: https://github.com/anthropics/claude-code/blob/main/CHANGELOG.md
  • Models overview: https://platform.claude.com/docs/en/about-claude/models/overview
  • Anthropic Academy courses page: https://anthropic.skilljar.com/
  • Anthropic Newsroom: https://www.anthropic.com/news

Conflict Labels

Use these labels in audits:

  • EXAM GUIDE CANONICAL: The official guide governs the answer.
  • PRODUCT UPDATED: Current Claude behavior or feature set has changed.
  • COURSE BROADER THAN EXAM: The course may be useful, but the topic is not exam-critical.
  • LOCAL DIGEST NEEDS UPDATE: A trainer file appears stale against the guide or official source.
  • INSUFFICIENT EVIDENCE: The claim may be true, but the companion cannot prove it from opened sources.

Exam Truth Vs Product Truth

When product information changes after the exam guide date, do not collapse the two.

Use this wording:

For exam prep, follow the official guide unless Anthropic has updated the exam guide itself.
For real-world Claude work, also note that the current product source says: [current fact].

4The 5 facts this course keeps repeating (exam gold)

1. The description is how Claude matches a skill to a request (semantic matching). Vague description means no trigger.
2. Skills load on demand. At startup only the name + description sit in context; the full body loads on match. Context-cheap by design.
3. Priority for same-named skills: Enterprise, Personal, Project, Plugins.
4. Each customization feature has its lane. CLAUDE.md (always-on), skills (request-driven), hooks (event-driven), subagents (isolated delegation), MCP (external tools). Don't force everything into skills.
5. Subagents inherit nothing. They don't see skills unless explicitly listed; built-in agents can't use skills at all.

TermPlain meaningExam nuance

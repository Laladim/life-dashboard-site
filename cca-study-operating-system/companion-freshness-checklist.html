Claude Freshness Checklist

Use this checklist when auditing whether a lesson note, trainer response, mock question, or study artifact may be stale against current Anthropic sources.

Claims That Need Freshness Checks

Check official Anthropic sources before accepting claims about:

  • Claude model names, availability, model IDs, or recommended model choices.
  • Context windows, output limits, beta headers, deprecations, or retired API behavior.
  • Claude Code commands, flags, hooks, plan mode behavior, output formats, or changelog items.
  • Tool use behavior, server tools, web search, code execution, web fetch, or tool result handling.
  • MCP features, MCP server configuration, resources, prompts, annotations, and security boundaries.
  • Claude app features, connectors, projects, artifacts, memory, enterprise controls, or role permissions.
  • Anthropic Academy recommended courses or course availability.
  • Pricing, rate limits, account policy, auth, billing, or admin features.

What To Flag

Flag as Critical if the stale claim could cause Shela to pick the wrong exam answer.

Flag as Moderate if the stale claim wastes study time, confuses surfaces, or changes a real-world Claude workflow.

Flag as Minor if the issue is wording or naming and does not affect exam judgment.

Flag as Context only if the current product update is interesting but not needed for the exam.

Required Freshness Language

When current product sources are checked, include:

Verified as of: YYYY-MM-DD
Source checked: [official source name]
Exam relevance: in scope / out of scope / context only

When current product sources are not checked, include:

Verdict: NEEDS LIVE VERIFICATION
Reason: This claim depends on current Anthropic product behavior.

Stale-Claim Examples

  • A lesson says an older Claude model is the latest model.
  • A mock question drills product trivia that the exam guide lists as out of scope.
  • A trainer response treats an Academy quiz topic as exam-required without guide support.
  • A Claude Code workflow references flags, paths, or behavior without checking current docs.
  • A note says schema guarantees factual truth instead of only output shape.

Freshness Does Not Override The Exam Guide

If a current Anthropic product source differs from the exam guide, mark both truths:

EXAM GUIDE CANONICAL: [exam-relevant statement]
PRODUCT UPDATED: [current product statement]
Recommended study action: Keep the exam answer guide-aligned, and add a real-world note separately.

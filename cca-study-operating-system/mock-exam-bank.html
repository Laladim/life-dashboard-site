Q01 - Domain 1 - Agentic Loop

A support agent calls a tool, receives the result, and must decide whether to continue. Which signal should the application use to know the model is finished?

A. The assistant says "I am done" B. A fixed maximum of 3 tool calls has been reached C. stop_reason == "end_turn" D. The answer looks complete to the user

Answer: C Why: Completion should be based on the model stop signal, not text parsing, guesses, or arbitrary caps.

Q02 - Domain 1 - Multi-Agent Orchestration

A research task has four independent document sets that can be analyzed separately and then combined. What design is best?

A. One long prompt that analyzes every document at once B. Parallel specialized subagents followed by synthesis C. A fixed sequential chain where each document waits for the previous one D. A customer-support escalation workflow

Answer: B Why: Independent specialized work is a strong fit for parallel subagents plus aggregation.

Q03 - Domain 1 - Subagent Context

A coordinator tells a subagent, "Use the customer issue we discussed earlier." The subagent gives a vague answer. What is the likely problem?

A. Subagents automatically inherit too much context B. Subagents inherit nothing unless the coordinator passes it explicitly C. The subagent should use sentiment analysis D. The subagent needs a larger model only

Answer: B Why: Subagents have isolated context; the prompt must include the needed facts.

Q04 - Domain 1 - Hooks vs Prompts

A refund agent must never process refunds above a policy limit without escalation. What is the safest design?

A. Add "please never do this" to the prompt B. Use a hook or code precondition that blocks the tool call C. Ask the model to rate its confidence D. Let the customer sentiment decide

Answer: B Why: Deterministic guarantees need code-level enforcement, not probabilistic prompt guidance.

Q05 - Domain 1 - Task Decomposition

A workflow always follows the same three dependent steps: extract, validate, then summarize. Which pattern fits best?

A. Prompt chaining B. Dynamic decomposition C. Random routing D. Unbounded multi-agent debate

Answer: A Why: Fixed dependent steps are a chain; each step relies on the previous output.

Q06 - Domain 1 - Session State

A long coding session has stale tool results and many unrelated turns. What is often safer than blindly resuming the whole transcript?

A. Start fresh from a structured summary and rerun needed lookups B. Trust the old transcript completely C. Increase temperature D. Hide the old tool results in the middle

Answer: A Why: A structured summary plus fresh lookups reduces stale-context risk.

Q07 - Domain 2 - Tool Description

Two tools have overlapping descriptions and Claude often picks the wrong one. What should you fix first?

A. Rename and clarify tool boundaries B. Add a separate classifier agent immediately C. Remove all tool descriptions D. Force every tool call manually

Answer: A Why: Tool descriptions drive selection. Overlap should be removed at the tool interface.

Q08 - Domain 2 - Structured Error

A lookup tool fails because the database is temporarily unavailable. What should it return?

A. An empty list B. A raw stack trace C. Structured error metadata with retryability and category D. A confident invented answer

Answer: C Why: The agent needs actionable error information to recover or escalate.

Q09 - Domain 2 - Tool Choice

The model must call some tool before answering, but the exact tool can vary. Which setting fits?

A. tool_choice: "auto" B. tool_choice: "any" C. No tools available D. Natural language reminder only

Answer: B Why: "any" forces a tool call while letting the model choose which tool.

Q10 - Domain 2 - MCP Triad

In MCP, which item represents passive reference context such as schemas or catalogs?

A. Tool B. Resource C. Prompt D. Hook

Answer: B Why: Resources provide passive context. Tools act. Prompts are reusable templates.

Q11 - Domain 2 - Built-In Tools

Claude needs to find files by filename pattern before reading them. Which tool concept matches that job?

A. Grep B. Glob C. JSON Schema D. Batch API

Answer: B Why: Glob finds paths/files. Grep searches text inside files.

Q12 - Domain 3 - CLAUDE.md Scope

A team standard is placed in a user's personal CLAUDE.md, and teammates do not follow it. What is wrong?

A. Team standards should live in project-level configuration B. User CLAUDE.md always syncs to everyone C. Slash commands replace all project rules D. The standard should be in a customer support tool

Answer: A Why: Shared team instructions must live in the repo/project scope.

Q13 - Domain 3 - Slash Commands vs Skills

Shela wants a reusable action she explicitly types when needed. Which feature is the best fit?

A. Slash command B. Skill C. Context window D. Sentiment trigger

Answer: A Why: Slash commands are manually invoked. Skills auto-load when relevant.

Q14 - Domain 3 - Path Rules

Different testing conventions apply only to *.test.tsx files. Where should this rule live?

A. A broad always-on instruction for every task B. A path-specific rule with matching globs C. A billing dashboard D. A confidence self-rating prompt

Answer: B Why: Path-specific rules save context and load only when relevant.

Q15 - Domain 3 - Plan Mode

Claude must migrate many files with architectural trade-offs. What workflow should be preferred?

A. Direct execution with no review B. Plan mode before changes C. Batch API only D. Ask the model for a confidence score

Answer: B Why: Plan mode gates larger ambiguous work before execution.

Q16 - Domain 3 - CI/CD

A CI job hangs because Claude is waiting for interactive input. What is the right fix?

A. Use non-interactive print mode B. Increase context window C. Add a refund tool D. Remove structured output

Answer: A Why: CI needs non-interactive execution.

Q17 - Domain 3 - Review Context

Claude wrote a patch and then reviews the same session's work. The review misses issues. What improves the review?

A. A fresh isolated review context B. The same session with more praise C. A higher temperature D. No tests in context

Answer: A Why: Independent context reduces self-review blind spots.

Q18 - Domain 4 - Explicit Criteria

A PR review prompt says "find bad code" and produces noisy comments. What is the best improvement?

A. Define exact report/ignore criteria with examples B. Ask for more comments C. Remove severity labels D. Trust the model to infer team standards

Answer: A Why: Concrete criteria reduce false positives and improve actionability.

Q19 - Domain 4 - Few-Shot

An extraction task has subtle categories the model confuses. What is usually most helpful?

A. Two to four targeted examples with rationale B. No examples, only a vague instruction C. A larger context window alone D. A billing-setting change

Answer: A Why: Few-shot examples teach the distinction and expected output.

Q20 - Domain 4 - Schema Shape

JSON Schema output validates successfully, but the extracted amount is wrong. What did the schema guarantee?

A. Truth of every value B. Output shape and types C. Source provenance D. Human approval

Answer: B Why: Schema enforces structure, not semantic truth.

Q21 - Domain 4 - Optional Fields

A document may not include a middle name. What schema design avoids fabrication?

A. Require middle_name always B. Make middle_name optional or nullable C. Ask Claude to invent missing data D. Hide the field from validation errors

Answer: B Why: Required fields can pressure the model to fabricate absent information.

Q22 - Domain 4 - Validation Retry

An extraction fails validation. What should the retry include?

A. Only "try harder" B. Original document, bad output, and exact validation errors C. A confidence score only D. A random new schema

Answer: B Why: Error-informed retry gives the model actionable correction context.

Q23 - Domain 4 - Batch Processing

A manager wants the cheapest option for an overnight audit report. Latency does not matter. What fits?

A. Batch processing B. Blocking real-time CI check C. Manual sentiment review D. User-level CLAUDE.md only

Answer: A Why: Batch is good for non-blocking, cost-sensitive work.

Q24 - Domain 4 - Multi-Pass Review

A large PR review misses cross-file issues. What design is better?

A. Per-file review plus separate integration pass B. One giant prompt only C. Never include file context D. Trust the first review completely

Answer: A Why: Focused passes reduce attention dilution and catch different issue types.

Q25 - Domain 5 - Critical Facts

A long support conversation summary loses dates and amounts. What should the system maintain separately?

A. A structured case-facts block B. A model confidence score C. A hidden larger font D. A generic "customer is upset" note

Answer: A Why: Critical numbers, dates, and sources should be preserved outside lossy summaries.

Q26 - Domain 5 - Escalation

Which is a reliable escalation trigger?

A. The customer explicitly asks for a human B. The model says it feels 60% confident C. The sentiment sounds negative D. A random timer expires

Answer: A Why: Explicit user request is a deterministic trigger; confidence and sentiment are unreliable.

Q27 - Domain 5 - Ambiguity

A lookup returns three possible customers with the same name. What should the agent do?

A. Guess the most likely one B. Ask for an identifier or disambiguating detail C. Process a refund for all three D. Ignore the ambiguity

Answer: B Why: Ambiguous matches should be clarified, not guessed.

Q28 - Domain 5 - Error Propagation

A search subagent cannot access one source but found partial results elsewhere. What should it do?

A. Hide the failure and synthesize anyway B. Abort the entire research task without details C. Return structured failure context plus partial results D. Invent coverage for the missing source

Answer: C Why: Coordinators need partial results and structured error context to recover or qualify output.

Q29 - Domain 5 - Large Codebase Context

Claude starts citing generic patterns instead of specific files in a long codebase session. What helps?

A. Delegate focused discovery to subagents and keep structured notes B. Continue adding unrelated context forever C. Increase temperature D. Remove file paths from prompts

Answer: A Why: Focused context and structured state help preserve specificity.

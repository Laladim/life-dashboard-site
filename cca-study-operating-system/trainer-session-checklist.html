CCA Trainer Session Checklist

Use this checklist every time the CCA trainer skill runs. Copy the template into progress.md or a session note when needed.

Pre-Session

  • [ ] Opened official guide or verified the question is covered by guide-derived materials.
  • [ ] Opened knowledge-base.md.
  • [ ] Opened progress.md.
  • [ ] Opened guide-mastery-matrix.md.
  • [ ] Opened commitments-tracker.md.
  • [ ] Picked one matrix row or one readiness gate for today.

Teaching

  • [ ] Explained one concept in plain English.
  • [ ] Gave one analogy.
  • [ ] Tied it to one official scenario.
  • [ ] Named the exam trap.
  • [ ] Named the correct design instinct.

Active Recall

  • [ ] Shela answered before seeing the explanation.
  • [ ] Drill type used: teach-back / flashcard / spot-the-trap / scenario classification / mock MCQ / build micro-task.
  • [ ] Result recorded: pass / partial / miss.
  • [ ] If missed, the trainer retaught differently and retried once.

Logging

  • [ ] Updated progress.md.
  • [ ] Updated guide-mastery-matrix.md status/evidence.
  • [ ] Updated commitments-tracker.md if a public commitment changed.
  • [ ] If readiness, artifact quality, source freshness, mock questions, or a Passed status was involved, ran or queued the CCA Trainer Companion audit.
  • [ ] Named next session's smallest target.

Done Gate

Do not say the session is done unless these are all true:

  • [ ] Source was checked.
  • [ ] One guide-backed concept was taught or reviewed.
  • [ ] Active recall happened.
  • [ ] The result was logged.
  • [ ] Any required companion audit was completed or explicitly logged as unresolved.
  • [ ] Next step was named.

How to Verify Key States

Permissions bypassed? Check the bottom statusline:

>> bypass permissions on (shift+tab to cycle)
All permission checks are skipped. No prompts will appear.
>> acceptEdits or >> default
Permissions NOT bypassed. Kill and restart with --dangerously-skip-permissions.

Mac caffeinated (won't sleep)? Detach from tmux, then run: ps aux | grep caffeinate

Running. Mac won't sleep.
Only see grep --color caffeinate. Not running. Fix: caffeinate -d &

Commands Reference
Start a New Session 
# Option 1: Use the startup script (recommended)
~/Life-Dashboard/.claude/start-telegram-session.sh

# Option 2: Manual start
cd ~/Life-Dashboard
tmux new -s claude-telegram
caffeinate -d & claude --channels "plugin:telegram@claude-plugins-official" --dangerously-skip-permissions
Session Management 
# Check if session is running
tmux ls

# Attach to see the live terminal
tmux attach -t claude-telegram

# Detach without killing (press in sequence)
Ctrl+B then D

# Kill the session (to restart fresh)
tmux kill-session -t claude-telegram

# Kill ALL tmux sessions
tmux kill-server
Inside Claude Code (when attached via tmux) 
# Clear context and start fresh
/clear

# Exit Claude Code entirely
/exit   or   Ctrl+C
System Commands 
# Prevent Mac from sleeping
caffeinate -d &

# Check if caffeinate is running
ps aux | grep caffeinate

# Check hook execution log
tail -20 ~/Life-Dashboard/.claude/hooks/hook-execution.log

# Check LaunchAgent status
launchctl list | grep claude

# Reload LaunchAgent
launchctl unload ~/Library/LaunchAgents/com.claude.telegram-session.plist
launchctl load ~/Library/LaunchAgents/com.claude.telegram-session.plist
Do's and Don'ts
Do's
  • Keep your Mac plugged in and lid open (or use caffeinate) when running remote sessions
  • Use Telegram for regular work (chatting, tasks, questions, session saves)
  • Use Termius only for session management (restart, clear context, check status)
  • Detach from tmux (Ctrl+B, D) before closing Termius
  • Check tmux ls first before starting a new session to avoid duplicates
  • Keep Tailscale running on both Mac and iPhone
  • Use /clear when context gets high (check statusline for %)
  • Save session before clearing via Telegram first
Don'ts
  • Don't close your Mac lid without caffeinate running
  • Don't run multiple Claude Telegram sessions at once
  • Don't share the bot token with anyone
  • Don't change access policy to "pairing" unless intended
  • Don't type sensitive passwords in Telegram chat
  • Don't expect message history in the bot
  • Don't force-quit Terminal while Claude is running
  • Don't use Termius for long conversations (use Telegram)
Tips
  • Bot can see photos sent on Telegram
  • Long responses auto-chunk at 4096 chars
  • All 10 behavioral hooks fire in Telegram session
  • All MCP servers available (Sheets, Slack, Beehiiv, Searchable, Keep)
  • LaunchAgent auto-starts session on Mac login
Troubleshooting
Bot Not Responding on Telegram
Cause: Claude Code session died or Mac went to sleep.
  1. Open Termius, connect to Lala Desktop
  2. Run tmux ls
  3. If no session: ~/Life-Dashboard/.claude/start-telegram-session.sh
  4. If exists but unresponsive: tmux kill-session -t claude-telegram then restart
  5. Check caffeinate: ps aux | grep caffeinate
Termius: "Connection Refused"
Cause: SSH not enabled or Tailscale not running.
  1. Tailscale connected on both devices?
  2. Mac: System Settings > General > Sharing > Remote Login = ON
  3. IP should be 100.104.9.62
  4. If IP changed: check login.tailscale.com
Termius: "Connection Timeout"
Cause: Mac is off, asleep, or Tailscale disconnected.
  1. Mac powered on and awake?
  2. Check Tailscale app on iPhone: Mac showing green/online?
  3. If offline, need physical access to wake it
Permission Prompts in Terminal
Cause: Session started without --dangerously-skip-permissions.
Statusline shows >> acceptEdits instead of >> bypass permissions on. 
tmux kill-session -t claude-telegram
cd ~/Life-Dashboard
tmux new -s claude-telegram

High Context (Statusline 70%+)
  1. Telegram: ask Claude to "save session" first
  2. Termius: tmux attach -t claude-telegram
  3. Type /clear inside Claude Code
  4. Or kill and restart for fully fresh start
